## Summary
- First production-ready release with hardened authentication, secure me-scoped APIs, and release governance automation.

## Changes
- Security:
  - Removed client-trusted identity for protected actions.
  - Enforced JWT-derived identity and role-based admin guard.
  - Deprecated legacy email-based endpoints with HTTP 410.
  - Added refresh-token rotation, logout token revoke, and auth audit logging.
- API:
  - Added me-scoped endpoints for social graph, notifications, and messages.
  - Added admin audit-log endpoint.
  - Added strict request validation with forbidden extra fields.
- Frontend:
  - Switched profile/messages/notifications to me-scoped authenticated APIs.
  - Added room-based chat UI and secure logout flow calling revoke endpoint.
- CI/CD:
  - Added backend test suite and API behavior tests.
  - Added coverage gate (>=60%), backend compile check, and frontend syntax checks.
  - Added release-gate workflow with release-note enforcement, optional Docker build/push, optional auto-deploy hooks, and optional notifications.
  - Added changelog automation during release.

## Migration Notes
- Deprecated endpoints now return HTTP 410:
  - GET /api/users/{email}/social
  - GET /api/notifications/{email}
  - GET /api/messages/{email}
- Replace with:
  - GET /api/users/me/social
  - GET /api/notifications/me
  - GET /api/messages/me
- Configure optional release secrets only if using advanced release automation:
  - DOCKERHUB_USERNAME, DOCKERHUB_TOKEN
  - RENDER_DEPLOY_HOOK_URL
  - RAILWAY_DEPLOY_HOOK_URL
  - RELEASE_NOTIFY_WEBHOOK_URL

## Validation
- CI status: backend tests and compile checks passing.
- Coverage status: backend coverage above enforced 60% threshold.
- Render smoke tests: documented in DEPLOYMENT_RUNBOOK.md and required via release gate input.
