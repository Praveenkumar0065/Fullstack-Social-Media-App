# Release Notes

Create one markdown file per release using this naming convention:

- release-notes/vX.Y.Z.md

Example:

- release-notes/v1.2.0.md

Minimum expected sections:

## Summary
- High-level overview of the release.

## Changes
- Security changes
- API changes
- Frontend changes
- CI/CD changes

## Migration Notes
- Any breaking changes or required action.

## Validation
- Mention smoke tests and CI gates passed.
