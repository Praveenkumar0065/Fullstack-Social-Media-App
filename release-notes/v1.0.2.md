## Summary
- This release improves authentication UX and robustness: login/signup screens were redesigned to a stronger split-panel experience, while backend signup/login validation and auth-route consistency were tightened.

## Changes
- Security:
  - Enforced strict email-format validation on auth request payloads by moving signup/login email fields to EmailStr.
  - Consolidated repeated authenticated-email extraction in API routes through a shared helper to reduce inconsistent auth checks.
- API:
  - Added focused test coverage for invalid-email signup rejection and growth/onboarding behavior.
  - Kept existing auth and social APIs intact with minimal behavior changes beyond stricter input validation.
  - Fixed DB bootstrap path that could trigger auth/login 500 under smoke checks when Mongo initialization executed.
- Frontend:
  - Refined login/signup interfaces to match the target split-panel visual design more closely.
  - Added/updated invite and onboarding flows and referral-aware signup handoff.
  - Added explicit feed retry state for failed initial loads.
- CI/CD:
  - Updated smoke test probe behavior to use a deterministic login probe email for rate-limit checks.
  - Synced APP_VERSION metadata to 1.0.2 in deployment and environment templates for consistent health/version checks.
  - Added specialized workspace custom agents for release, security, fullstack maintenance, and UX polish workflows.

## Migration Notes
- None.
- Operational note: clients sending invalid email strings to auth endpoints will now receive validation errors (422) instead of being accepted.
