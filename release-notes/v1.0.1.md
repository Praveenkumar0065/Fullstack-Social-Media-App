## Summary
- Patch release that finalizes production hardening and test stability on top of v1.0.0.

## Changes
- Security and Auth:
  - Introduced PBKDF2-SHA256 password hashing for stored credentials.
  - Added backward-compatible password verification and automatic migration path from legacy plaintext records after successful login.
  - Updated seed/admin credential storage to hashed format.
- Monitoring and Observability:
  - Added optional Sentry initialization with environment-controlled DSN and trace sample rate.
  - Kept runtime safe when Sentry SDK is not installed (non-fatal fallback).
- Frontend Performance:
  - Added route-level lazy loading for major pages.
  - Added Suspense fallback for chunk loading.
  - Added manual chunk strategy in Vite build for improved caching and startup performance.
- Deployment and Operations Docs:
  - Added comprehensive global launch guide for Render + Vercel, custom domains, env configuration, and final validation.
  - Updated environment templates for auth iteration tuning and monitoring settings.
- Quality:
  - Fixed backend API behavior test to align with authenticated endpoint requirements.
  - Backend tests are now fully green (11 passing).

## Migration Notes
- New optional backend env vars:
  - AUTH_PASSWORD_ITERATIONS
  - SENTRY_DSN
  - SENTRY_TRACES_SAMPLE_RATE
  - APP_ENV
- Frontend optional env var:
